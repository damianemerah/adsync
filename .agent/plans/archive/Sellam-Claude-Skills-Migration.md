# Sellam — Claude Skills Migration: Campaign Copy Generation

## Implementation Plan v1.1

> **Scope:** Migrate campaign copy and strategy generation from OpenAI (`gpt-4o-mini`) to Anthropic Claude using the Skills API.
> **Out of scope:** Image prompt generation (stays on OpenAI for now — `FLUX_AD_GENERATOR_SYSTEM` and `FLUX_EDIT_SYSTEM` in `prompts.ts` are untouched).
> **Primary goal:** Higher-quality, vertically-specialized Nigerian ad copy that enforces a proven high-converting Facebook/Meta structure.

> **v1.1 Architecture Decision:** Claude is the single source of truth for all AI decisions — intent classification, vertical detection, refinement question priority, and copy generation. The shell (`audience-chat-step.tsx`) is a pure renderer that reacts to `meta` fields Claude returns. No parallel decision-making between prompt and shell.

---

## Context: Why This Matters

The current system prompt in `src/lib/ai/prompts.ts` is a monolithic 3,500-token document sent **in full on every single API call**, regardless of whether the user sells shawarma or skincare. It has no enforced copy structure — only tone rules and word bans.

The Claude Skills architecture fixes three things at once:

1. **Only loads relevant vertical copy rules** — fashion campaign doesn't load food copy rules.
2. **Enforces a proven copy skeleton** (Hook → Benefit → Proof → CTA) on every generation.
3. **Enables vertical-specific WhatsApp prefill templates** instead of the current generic message.

---

## Proven High-Converting Facebook Ad Copy Structure

**Every primary text generated by Sellam must follow this skeleton.** This is non-negotiable and must be baked into both the core skill and every vertical skill.

```
LINE 1 — HOOK (max 12 words)
Stop the scroll. Emotional, specific, Nigerian voice.
No vague openers. Lead with price (budget), exclusivity (premium), or a sharp question (mid).

LINES 2–3 — BENEFIT / SOLUTION
What they get + what makes this different.
One concrete differentiator: speed, price, material quality, convenience, location.
Keep it Nigerian and tangible. "Perfect for owambe, office, or date night."

LINE 4 — PROOF (MANDATORY — no exceptions)
One credible social signal or urgency cue.
  - If user gave real numbers → use them exactly.
  - If no proof given → use safe regional framing ONLY:
      ✅ "Loved by customers across Lagos Island and Mainland"
      ✅ "Popular choice for Abuja owambe season"
      ✅ "Only a few pieces left this week"
      ❌ NEVER fabricate numbers ("Over 500 orders") unless stated by user.

FINAL LINE — CTA (direct action, tells buyer what happens next)
  - WhatsApp: "Send us a message — we reply with prices and delivery info in minutes"
  - Buy now:  "Tap to shop — delivered to your door in Lagos"
  - Book:     "WhatsApp us to check availability and book your slot"
```

**What to NEVER write:**

- "Elevate your..."
- "Experience the difference"
- "Discover the joy of..."
- Any claim without a concrete anchor
- Vague quality claims with no specifics

## Architecture Overview

```
Current Architecture:
User input → /api/ai/generate → service.ts → OpenAI (gpt-4o-mini) + monolithic ADS_SYSTEM_PROMPT → JSON
                                             ↑
                               Shell pre-classifies intent
                               Shell detects vertical via regex
                               Shell decides which questions to ask
                               (two decision-makers, can drift)

New Architecture:
User input → /api/ai/generate → service.ts (Claude) → Skills Container
                                                          ├── core-strategy-ng   (always loaded)
                                                          ├── copy-fashion-ng     (always passed as candidate)
                                                          ├── copy-food-ng        (always passed as candidate)
                                                          ├── copy-beauty-ng      (always passed as candidate)
                                                          ├── copy-services-ng    (always passed as candidate)
                                                          └── policy-guard-ng     (always passed as candidate)
                                                        → JSON (AIStrategyResult — same shape as before)
                                                          ↓
                               Shell receives meta fields and RENDERS decisions.
                               Shell makes NO independent AI decisions.
```

**How vertical selection works:** All skills are passed as candidates on every call. Claude reads each skill's YAML `description` frontmatter and auto-loads only the ones relevant to the user's input. A shawarma seller triggers `copy-food-ng`. A boutique triggers `copy-fashion-ng`. A hybrid business ("I sell wigs and teach haircare workshops") can trigger both `copy-beauty-ng` and `copy-services-ng`. No regex, no mapping dictionary, no pre-classification function in the shell.

**Responsibility boundary — Claude owns:**

- Intent classification (strategy / question / refinement / image request / image edit / copy correction)
- Vertical detection (via Skills auto-trigger on YAML descriptions)
- Whether and which refinement question to ask (STEP 9 in core skill)
- All copy generation, structure enforcement, and WhatsApp prefill selection

**Responsibility boundary — Shell owns:**

- Objective mismatch detection (`lastGeneratedObjective` vs current `objective`) — this is UI state, not an AI decision
- Sentinel handling (`__OBJECTIVE_REWRITE__`, `__OBJECTIVE_KEEP__`) — UI flow controls
- Container session management (`containerId` passing for multi-turn)
- Network error detection and retry UI
- Rendering all `meta` fields Claude returns — it does not override them

**Models:**

- `claude-sonnet-4-6` → Initial campaign strategy generation (`generateAndSaveStrategy`)
- `claude-haiku-4-5-20251001` → Copy refinements (`handleCopyRefinement` calls)

**OpenAI stays for:** All image generation (`ai-images.ts`, `FLUX_AD_GENERATOR_SYSTEM`, `FLUX_EDIT_SYSTEM`). Do not touch those files.

### Skill 1: `core-strategy-ng` (Always Loaded)

**File:** `src/lib/ai/skill-definitions/core-strategy-ng/SKILL.md`

This replaces the bulk of `ADS_SYSTEM_PROMPT` **and** absorbs all intent classification logic previously handled by `classifyUserInput()` in `audience-chat-step.tsx`. The shell will no longer make any independent classification decisions — it simply renders what this skill instructs Claude to return in the `meta` object.

**What to include in this skill:**

**A — Intent Classification (moved from shell's `classifyUserInput`)**

The skill must teach Claude to silently classify every user input into one of these types before doing anything else. These map directly to `meta.input_type` in the JSON output.

```
TYPE_A: Any description of a product, service, or business → generate full strategy immediately
TYPE_B: Single bare word / price only / location only → set needs_clarification: true, ask one unlock question
TYPE_C: Advertising question (how much to spend, which platform, etc.) → set is_question: true, answer in question_answer
TYPE_D: Copy refinement instruction ("make it shorter", "more fire", "add urgency") → update copy only, do not rebuild strategy
TYPE_E: Conversational confirmation ("thanks", "looks good", "are we done") → set is_question: true, confirm politely
TYPE_F: Structural image edit request ("add a model", "replace background", "use mannequin") → set is_question: true, instruct user to use Studio
TYPE_G: Product fact correction ("it's not for men", "we don't deliver", "wrong price") → rebuild copy with corrected fact, also return updated interests
TYPE_H: Image generation request ("generate image", "design the ad", "no street background") → set is_question: true, instruct shell to trigger image generation
```

Include the exact same keyword and pattern examples from the current `classifyUserInput` function so Claude's classification matches what the shell previously did:

- Question indicators: "how much", "how do", "which platform", "can I", "should I", "?", "explain", "what is"
- Refinement indicators: "make it", "shorter", "longer", "more urgent", "try again", "rewrite", "make am fire", "e too long"
- Image edit indicators: "add to the image", "replace with", "put a model", "use as stand", "mannequin"
- Correction indicators: "it's not", "actually", "wrong", "we don't", "remove mention of", "don't mention"
- Image generation indicators: "generate image", "create image", "design the ad", "no background", "use background"

**B — Refinement Question Logic (single source of truth, replaces STEP 9)**

Claude must ask at most ONE refinement question per response. Priority order when input is ambiguous:

```
1. Product/service clarity — if it's genuinely unclear what they sell
2. Delivery scope — "Do you sell in-store only or do you also ship? Changes your reach."
3. Price tier — "Are your prices budget-friendly, mid-range, or premium? Helps target the right buyers."
4. Gender — only ask if genuinely ambiguous (not if product name implies gender clearly)
5. Active offer — "Do you have a promo or discount I can use in the copy?"
```

Ask only when: confidence < 0.75 AND the missing dimension would materially change targeting or copy angle.
Set `refinement_question: null` when input already covers product + location + audience clearly.

**IMPORTANT:** This means STEP 9 in the current `ADS_SYSTEM_PROMPT` is the ONLY place refinement questions are decided. The shell will render `meta.refinement_question` as-is and will NOT apply any additional question logic of its own.

**C — Everything else from the current `ADS_SYSTEM_PROMPT` STEP 0–6 and STEP 8:**

- STEP 0: Input classification (now expanded to TYPE_A through TYPE_H above)
- STEP 1: Inference rules (gender, location, price tier, business type)
- STEP 2: Pidgin + informal language normalization
- STEP 3: Location normalization
- STEP 4: Business type → targeting matrix
- STEP 4B: Behavior targeting (sales-optimized)
- STEP 5: Multi-product handling
- STEP 6: CTA intent
- STEP 8: Plain English summary format
- OUTPUT FORMAT: The exact `AIStrategyResult` JSON schema
- OUTPUT RULES: Return only raw JSON, no markdown, no backticks

**Remove** the old STEP 7 (copy rules) — that moves entirely to vertical skills.

**YAML frontmatter:**

```yaml
---
name: core-strategy-ng
version: "1.0.0"
description: >
  Core campaign strategy skill for Nigerian and West African SMEs on Sellam.
  Always load this skill. Owns ALL intent classification, refinement question
  priority, audience targeting, location normalization, Pidgin language parsing,
  CTA intent mapping, and JSON output schema. Load alongside vertical copy skills
  which Claude will auto-select based on business type.
---
```

## Part 2: Skill Upload Script

**File:** `src/scripts/upload-skills.ts`

This script runs once (and again on any skill update) to upload all skill definitions to Anthropic and save the returned IDs to your `.env`.

```typescript
/**
 * upload-skills.ts
 * Run with: npx ts-node src/scripts/upload-skills.ts
 *
 * Reads all skill folders from src/lib/ai/skill-definitions/,
 * uploads each to Anthropic's Skills API, and prints the env vars
 * to paste into your .env file.
 *
 * Requires: ANTHROPIC_API_KEY in environment before running.
 */

import Anthropic from "@anthropic-ai/sdk";
import fs from "fs";
import path from "path";

const anthropic = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });

const SKILLS_DIR = path.join(__dirname, "../lib/ai/skill-definitions");

async function uploadSkill(
  skillDir: string,
): Promise<{ name: string; id: string }> {
  const skillName = path.basename(skillDir);
  const skillMdPath = path.join(skillDir, "SKILL.md");

  if (!fs.existsSync(skillMdPath)) {
    throw new Error(`No SKILL.md found in ${skillDir}`);
  }

  const skillContent = fs.readFileSync(skillMdPath, "utf-8");

  // Collect any additional markdown files in the skill folder
  const additionalFiles = fs
    .readdirSync(skillDir)
    .filter((f) => f.endsWith(".md") && f !== "SKILL.md")
    .map((f) => ({
      name: f,
      content: fs.readFileSync(path.join(skillDir, f), "utf-8"),
    }));

  // Upload to Anthropic Skills API
  // NOTE: Check Anthropic docs for exact API shape at time of implementation:
  // https://docs.anthropic.com/en/docs/build-with-claude/skills
  const response = await (anthropic as any).beta.skills.create({
    name: skillName,
    files: [{ name: "SKILL.md", content: skillContent }, ...additionalFiles],
  });

  return { name: skillName, id: response.id };
}

async function main() {
  const skillDirs = fs
    .readdirSync(SKILLS_DIR)
    .map((d) => path.join(SKILLS_DIR, d))
    .filter((d) => fs.statSync(d).isDirectory());

  console.log(`Found ${skillDirs.length} skills to upload...\n`);

  const results: { name: string; id: string }[] = [];

  for (const dir of skillDirs) {
    const skillName = path.basename(dir);
    process.stdout.write(`Uploading ${skillName}... `);
    try {
      const result = await uploadSkill(dir);
      results.push(result);
      console.log(`✅ ${result.id}`);
    } catch (err) {
      console.log(`❌ Failed: ${err}`);
    }
  }

  console.log("\n--- Paste these into your .env file ---\n");
  for (const { name, id } of results) {
    const envKey = `SKILL_ID_${name.toUpperCase().replace(/-/g, "_")}`;
    console.log(`${envKey}=${id}`);
  }
}

main();
```

// ADD at top of file (keep existing OpenAI import)
import Anthropic from "@anthropic-ai/sdk";
import { AIInput, AIStrategyResult } from "./types";

// ADD: Anthropic client
const anthropic = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY!,
  timeout: 30000, // Skills container adds ~1-2s overhead vs OpenAI
  maxRetries: 1,
});

// ADD: Skill ID constants — set by upload-skills.ts, stored in .env
// This is the COMPLETE skill configuration for this file. Nothing else needed.
const SKILL_IDS = {
  coreStrategy: process.env.SKILL_ID_CORE_STRATEGY_NG!,
  fashion: process.env.SKILL_ID_COPY_FASHION_NG!,
  food: process.env.SKILL_ID_COPY_FOOD_NG!,

  services: process.env.SKILL_ID_COPY_SERVICES_NG!,
  policyGuard: process.env.SKILL_ID_POLICY_GUARD_NG!,
};

// ADD: Build skill list — ALL skills passed as candidates on every call.
// DO NOT add vertical detection logic here. DO NOT filter this list by
// business type, keywords, or industry. Pass all of them every time.
// Claude reads each skill's YAML description and auto-loads only the ones
// relevant to the user's input. Unloaded skills cost no tokens.
function buildSkillList(): any[] {
  return [
    { type: "custom", skill_id: SKILL_IDS.coreStrategy, version: "latest" },
    { type: "custom", skill_id: SKILL_IDS.fashion, version: "latest" },
    { type: "custom", skill_id: SKILL_IDS.food, version: "latest" },
    { type: "custom", skill_id: SKILL_IDS.beauty, version: "latest" },
    { type: "custom", skill_id: SKILL_IDS.services, version: "latest" },
    { type: "custom", skill_id: SKILL_IDS.policyGuard, version: "latest" },
  ];
}

// ADD: Build the user message.
// This is the raw input Claude receives. No pre-processing, no classification,
// no intent routing happens before this. Claude's core-strategy-ng skill
// owns all of that logic internally.
function buildUserMessage(input: AIInput): string {
  return [
    `Business: ${input.businessDescription}.`,
    `Location: ${input.location || "Nigeria"}.`,
    input.industry ? `Industry: ${input.industry}.` : "",
    input.sellingMethod ? `Selling Method: ${input.sellingMethod}.` : "",
    input.priceTier ? `Price Tier: ${input.priceTier}.` : "",
    input.customerGender ? `Target Audience: ${input.customerGender}.` : "",
    input.objective
      ? `\nCAMPAIGN OBJECTIVE: ${input.objective}\nTone: ${input.objectiveContext?.tone || "Standard"}\nTargeting Bias: ${input.objectiveContext?.targetingBias || "Standard"}\nPreferred CTA: ${input.objectiveContext?.ctaBias || "Standard"}`
      : "",
    input.currentCopy
      ? `\nCURRENT COPY TO REFINE (edit this, do not rebuild from scratch):\nHeadline: "${input.currentCopy.headline}"\nBody: "${input.currentCopy.primary}"\nApply the refinement instruction above. Keep what works.`
      : "",
  ]
    .filter(Boolean)
    .join("\n");
}

// ADD: Main Claude generator — Sonnet for initial strategy generation
async function generateWithClaude(input: AIInput): Promise<AIStrategyResult> {
  const response = await anthropic.beta.messages.create({
    model: "claude-sonnet-4-6",
    max_tokens: 2048,
    betas: ["code-execution-2025-08-25", "skills-2025-10-02"],
    container: {
      skills: buildSkillList(),
    },
    tools: [{ type: "code_execution_20250825", name: "code_execution" }],
    messages: [{ role: "user", content: buildUserMessage(input) }],
  } as any); // `as any` until Anthropic SDK types are fully updated for Skills

  const textBlock = response.content.find((c: any) => c.type === "text");
  if (!textBlock || !("text" in textBlock)) {
    throw new Error("No text response from Claude Skills");
  }

  // Strip code fences if Claude wraps JSON (defensive — core skill instructs against this)
  const raw = (textBlock as any).text
    .replace(/```json\n?/g, "")
    .replace(/```\n?/g, "")
    .trim();

  return JSON.parse(raw) as AIStrategyResult;
}

// ADD: Copy refinement — Haiku for iterative edits (faster + cheaper than Sonnet)
// Called when the route receives a refinementInstruction alongside existing copy.
export async function refineAdCopyWithClaude(
  input: AIInput,
  refinementInstruction: string,
): Promise<AIStrategyResult> {
  const response = await anthropic.beta.messages.create({
    model: "claude-haiku-4-5-20251001",
    max_tokens: 1024,
    betas: ["code-execution-2025-08-25", "skills-2025-10-02"],
    container: { skills: buildSkillList() },
    tools: [{ type: "code_execution_20250825", name: "code_execution" }],
    messages: [
      {
        role: "user",
        content: `${buildUserMessage(input)}\n\nRefinement instruction: ${refinementInstruction}`,
      },
    ],
  } as any);

  const textBlock = response.content.find((c: any) => c.type === "text");
  if (!textBlock || !("text" in textBlock))
    throw new Error("No response from Claude Haiku");

  const raw = (textBlock as any).text
    .replace(/```json\n?/g, "")
    .replace(/```\n?/g, "")
    .trim();

  return JSON.parse(raw) as AIStrategyResult;
}
