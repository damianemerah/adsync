## Design Context

### Users

Nigerian SME owners — fashion sellers, food vendors, beauty businesses, and service providers. B2C & B2B included. They sell **primarily via WhatsApp**, are phone-first, non-technical, and measure success in chats received and sales closed — not impressions or CPM. They are hustlers. The product must feel like a tool built _for them_, not adapted down from a Western SaaS product.

- **Primary device**: Android or iPhone, 375–390px viewport
- **Context of use**: On the go, between orders, often on a slow connection
- **Job to be done**: Launch an ad quickly → get WhatsApp messages → record sales → know which ad worked
- **Emotional goal**: Confidence that their money is working for them

### Brand Personality

**Crisp · Confident · Nigeria-First**

Three-word brand character: **Precise. Empowering. Real.**

- Not corporate, not startup-cute — grounded and serious like a tool you trust with your money
- Energetic but disciplined — the emerald green is bold; everything else is restrained
- WhatsApp-native context should feel familiar, not foreign

### Aesthetic Direction

**"Stripe meets Lagos"** — the discipline and structural precision of Stripe/Linear/Vercel, combined with the energy and confidence of a Nigerian SME category leader.

- **Flat UI**: Depth is created by 1px `border-border` lines and background shifts, NOT floating shadows
- **High-contrast**: Deep navy (#030018) backgrounds on landing; clean white (#ffffff) on dashboard
- **Single accent dominance**: Emerald green is the **only** thing that pops. Everything else defers.
- **Dark mode**: Fully supported via `.dark` class — all tokens have dark variants

**Visual references**: Stripe Dashboard, Linear, Vercel — the "crisp modern" category  
**Anti-references**: Rounded bubbly SaaS (Notion-style), glassmorphism, neon-on-dark, gradient text on metrics

### Design Principles

1. **Flat Over Floaty** — Borders create structure. Shadows create noise. Never use `shadow-lg` or `shadow-md` on cards.
2. **Emerald Leads** — Primary CTA (`bg-primary`) is the ONE thing that pops. Secondary and tertiary actions must be visually deferential.
3. **Mobile Is Canon** — 375px is the real screen size. Every layout decision is mobile-first. Never hide critical functionality on mobile.
4. **Token Purity** — Zero hex codes in JSX. All values via semantic CSS custom properties. No arbitrary Tailwind values.
5. **AI Is Always Purple** — Every AI-powered feature uses `text-ai` / `bg-ai` (Violet, #8b5cf6). No exceptions — not green, not blue.

### Text Color Decision Tree

```
Is it a heading or primary body copy?     → text-foreground
Is it a description, icon, or sidebar?   → text-subtle-foreground
Is it a placeholder or disabled state?   → text-muted-foreground
Is it the primary brand action?          → text-primary
Is it an AI-powered feature?             → text-ai
Is it an error?                          → text-destructive
```

> ⚠️ **Never use `text-muted-foreground` for text you want people to actually read.** It is only for placeholders, disabled states, and purely decorative metadata.

### Typography Tokens

| Role                  | Font                  | Class          | Weight                |
| --------------------- | --------------------- | -------------- | --------------------- |
| Headings (h1, h2, h3) | Montserrat Alternates | `font-heading` | 700                   |
| All body text         | Montserrat            | `font-sans`    | 400 / 500 / 600 / 700 |

**RULES**:

- ❌ Do NOT use Inter, Roboto, Open Sans, system-ui, or any other font — the font stack is locked
- `font-heading` on all H1–H3, with `letter-spacing: -0.025em` (applied globally)
- Body, labels, links: `font-sans`

### Shape & Radius

| Element               | Class                 |
| --------------------- | --------------------- |
| Cards, containers     | `rounded-lg` (0.5rem) |
| Inputs, badges, small | `rounded-md`          |
| Pills (tags only)     | `rounded-full`        |

**BANNED**: `rounded-2xl`, `rounded-3xl` — they violate the Crisp Modern flat aesthetic.

### Shadows

| Situation                    | Rule                                              |
| ---------------------------- | ------------------------------------------------- |
| Cards, panels, surfaces      | ❌ No shadow — use `border border-border` instead |
| Floating dropdowns, tooltips | `shadow-sm` + `border border-border`              |
| `shadow-md`, `shadow-lg`     | ❌ Never — too floaty, breaks the flat aesthetic  |

## Strict Anti-Patterns (Tenzu-Specific)

These are violations that must be flagged P1 or higher in any `/audit` or `/normalize` pass:

| Violation                                   | Correct Fix                                   | Severity |
| ------------------------------------------- | --------------------------------------------- | -------- |
| `text-gray-600`, `text-slate-500`           | `text-subtle-foreground`                      | P1       |
| `bg-blue-500`, `bg-[#123]`                  | Use semantic token or add to `globals.css`    | P0       |
| `shadow-md`, `shadow-lg` on cards           | `border border-border`                        | P1       |
| `rounded-3xl`, `rounded-2xl` on containers  | `rounded-lg`                                  | P2       |
| `import { X } from 'lucide-react'`          | `import { X } from 'iconoir-react'`           | P1       |
| `text-purple-500` for AI features           | `text-ai`                                     | P1       |
| AI feature NOT using `text-ai`/`bg-ai`      | Add `text-ai`                                 | P1       |
| Hardcoded `₦` or `NGN` string               | Derive from `org.currency_default`            | P0       |
| Multiple primary buttons in same view       | Demote secondary actions to `variant="ghost"` | P2       |
| `text-muted-foreground` on readable content | `text-subtle-foreground`                      | P2       |
| Glassmorphism, backdrop-blur decorative     | Remove — not Tenzu aesthetic                  | P2       |
| Gradient text on metrics/headings           | Plain `text-foreground`                       | P2       |
| `await` on analytics/notification calls     | Fire-and-forget `.then()` only                | P1       |

## Accessibility Requirements

- Text contrast ratio ≥ 4.5:1 (WCAG AA)
- All interactive elements have `:focus-visible` states via `ring-ring`
- No semantic information conveyed by color alone
- Touch targets ≥ 44×44px (mobile-first users)
